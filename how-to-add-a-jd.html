How to add a Job Description

A short, no-jargon guide. You just drag a file and click a button — it takes about a minute.

⏱ ~1 minute  ·  no technical skills needed
🔗 The library lives here: ant-jd-repository.github.io
One-time setup. You need a free GitHub account, and to be added to the project. If Step 1 below asks you to sign in, sign in at github.com/login. If it says “404” or “you don’t have access”, ask Gabriel to invite you to the Ant-JD-Repository project — it takes him 10 seconds.
1

Open the upload page

Click the button below. It opens the exact page where the Job Descriptions live — you don’t have to look for it. Tip: save it as a browser bookmark for next time.

Open the upload page →
2

Add your file

Drag the Job Description file (a Word .docx file) from your computer and drop it onto that page. Or click “choose your files” and pick it.

The file name doesn’t matter. The role title, business unit and location are read from inside the document.

3

Save it

Scroll to the bottom of the page and click the green “Commit changes” button. You can leave the little description box empty.

4

See it live

Wait about one minute (the site updates itself), then open the library and refresh the page. Your Job Description is now there, searchable by Position, Business Unit and Location.

Open the library →

Refreshing: on a Mac press Cmd+R (Windows: Ctrl+F5). On Safari, please don’t use Cmd+Shift+R — that opens Reader view.

🔄 Update a JD

Upload the new version the same way (Steps 1–3). If it has the same file name, it replaces the old one.

🗑️ Remove a JD

Open the file in the jds folder on GitHub, click the trash icon, then Commit changes. It disappears from the site shortly after.

Good to know

  • Please use the standard Ant JD template (v2.x) so the details are read correctly.
  • Duplicates are caught automatically. If you upload a file that is exactly the same as one already there, it won’t be shown twice — and a small yellow notice on the site tells you which file was the duplicate. (If even one word is different, it counts as a new, separate JD.)
  • The update only runs when you actually add or change a file — nothing runs on an empty day.
